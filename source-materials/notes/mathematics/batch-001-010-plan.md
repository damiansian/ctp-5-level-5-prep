# Mathematics Batch 001-010 Plan

## Goal

Start with Questions 1-10 and avoid treating the whole Mathematics unit like one giant problem.

## Recommended Split

### Likely ready for direct import

- `2` prime number
- `3` comparison symbol
- `5` probability of two heads
- `6` pounds to ounces
- `8` marble probability
- `9` exponent notation
- `10` right triangle properties

These look mostly text-first and should move quickly.

### Likely needs figure-aware handling

- `1` area of a rectangle
- `4` shape with one pair of perpendicular lines
- `7` volume from base area and height

These are not impossible, but they depend more directly on visual presentation.

## Practical Recommendation

The fastest calm start is:

1. Import `2, 3, 5, 6, 8, 9, 10` first.
2. Add `1, 4, 7` once we decide how we want to handle figure-dependent stimuli.

## Why This Split Works

- It gives us early momentum.
- It keeps the first math batch from getting blocked on graphics.
- It lets us test how well the current app handles math language before we introduce diagrams.

## Figure Notes For Later

- `1` can probably be represented temporarily with a short text description of the rectangle dimensions.
- `4` may need either cropped shape images or a very careful text fallback.
- `7` may be text-usable because the base area and height are given numerically, even though it references a prism.

## Suggested Next Move

Create a normalized draft for the direct-import set:

- Questions `2, 3, 5, 6, 8, 9, 10`

Then verify those against the OCR PDF before adding them to the app.
